# ROOT CAUSE IDENTIFIED AND FIXED

**Date:** October 28, 2025 (continued session)
**Issue:** All 12 claims showing "Only 0 source(s) found" - evidence retrieval completely failing

## THE ACTUAL BUG (Not logging, not code loading)

**Location:** `backend/app/pipeline/retrieve.py` line 85-90

**Problem:**
```python
# BROKEN CODE (retrieve.py was calling with unsupported parameters):
evidence_snippets = await self.evidence_extractor.extract_evidence_for_claim(
    claim_text,
    max_sources=self.max_sources_per_claim * 2,
    context=context,                # ← Parameter NOT supported by evidence.py
    claim_type=claim_type           # ← Parameter NOT supported by evidence.py
)
```

**But evidence.py signature is:**
```python
async def extract_evidence_for_claim(self, claim: str, max_sources: int = 5) -> List[EvidenceSnippet]:
```

**Result:**
- TypeError raised: "got unexpected keyword arguments 'context', 'claim_type'"
- Exception caught in line 47-49 of retrieve.py
- Returned empty list `[]` for ALL claims
- All claims ended up with 0 sources → insufficient_evidence

## THE FIX

Removed unsupported parameters from retrieve.py call:
```python
# FIXED CODE:
evidence_snippets = await self.evidence_extractor.extract_evidence_for_claim(
    claim_text,
    max_sources=self.max_sources_per_claim * 2
)
```

**Why This Happened:**
When I reverted the failed claim_type_detector implementation, I removed `context` and `claim_type` parameters from evidence.py, but retrieve.py was STILL calling with those parameters. The TypeError was silently caught and logged as an exception, but that log never appeared because logging wasn't configured properly.

**Expected Result After Fix:**
- Evidence retrieval should work again
- Should return to ~50/100 baseline score
- Claims should have 3-5 sources each
